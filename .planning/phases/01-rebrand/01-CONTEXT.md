# Phase 1: Rebrand - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Rename every SpoilerShield artifact to Veil: UI strings, extension manifest, localStorage keys, chrome.storage keys, postMessage types, custom DOM events, console logs, and icon files. No new functionality. No data loss on upgrade for existing users.

Supabase edge function names and VITE_* env var names are explicitly out of scope.

</domain>

<decisions>
## Implementation Decisions

### Icon approach
- Generate PNG icons from `extension/icons/veil-icon.svg` using a script (not manual export)
- Output: `icon16.png`, `icon48.png`, `icon128.png` — overwrite existing files in place
- Manifest icon references stay unchanged (same filenames)

### localStorage migration
- One-time migration at startup: read old `spoilershield-*` keys, write to new `veil-*` keys, then delete old keys
- Safe order: write new keys first, delete old keys after — worst case is duplicate data, never data loss
- Extends the existing legacy migration pattern already in `useSessionStore.ts`
- Keys to migrate:
  - `spoilershield-sessions` → `veil-sessions`
  - `spoilershield-active-session` → `veil-active-session`
  - `spoilershield-msgs-{id}` → `veil-msgs-{id}` (enumerate all matching keys)
  - `spoilershield-chat` → already handled by legacy migration; leave as-is

### Rename scope
- Rename everything: UI strings, manifest, storage keys, console.log strings (`[SpoilerShield]` → `[Veil]`), code comments, and internal variable names
- Full clean codebase — no SpoilerShield references anywhere after this phase

### postMessage and custom events
- `SPOILERSHIELD_SHOW_INFO` → `VEIL_SHOW_INFO`
- `SPOILERSHIELD_CONTEXT` → `VEIL_CONTEXT`
- `SPOILERSHIELD_REQUEST_SHOW_INFO` → `VEIL_REQUEST_SHOW_INFO`
- `SPOILERSHIELD_REQUEST_CONTEXT` → `VEIL_REQUEST_CONTEXT`
- `SPOILERSHIELD_REQUEST_REDETECT` → `VEIL_REQUEST_REDETECT`
- Custom DOM event `spoilershield-messages-updated` → `veil-messages-updated`
- Must update all three sides: content.js, sidepanel.js, React app (useSidePanel hook + Index.tsx)

### chrome.storage keys
- `spoilershield_show_info` → `veil_show_info`
- `spoilershield_context` → `veil_context`
- Update in content.js (writes) and sidepanel.js (reads + watches)

### Supabase edge functions — NOT renamed
- `spoiler-shield-chat`, `classify-question`, `audit-answer` etc. keep their names
- VITE_SUPABASE_URL / VITE_SUPABASE_PUBLISHABLE_KEY env vars keep their names
- These are infrastructure, not user-facing

### Claude's Discretion
- PNG generation tool/library choice (sharp, canvas, rsvg-convert, or other)
- Exact wording of updated console.log messages beyond `[Veil]` prefix

</decisions>
