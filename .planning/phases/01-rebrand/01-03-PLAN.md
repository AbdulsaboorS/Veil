<task type="auto">
  <name>Task 1: Generate Veil PNG icons from veil-icon.svg using sips</name>
  <files>extension/icons/icon16.png, extension/icons/icon48.png, extension/icons/icon128.png</files>
  <action>
Use macOS sips (available at /usr/bin/sips) to rasterize veil-icon.svg into three PNG icon sizes. Run these commands from the repo root:

```bash
# Generate 128px first (highest quality source for downscaling)
sips -s format png --resampleWidth 128 extension/icons/veil-icon.svg --out extension/icons/icon128.png

# Downscale to 48px and 16px from the 128px PNG
sips -z 48 48 extension/icons/icon128.png --out extension/icons/icon48.png
sips -z 16 16 extension/icons/icon128.png --out extension/icons/icon16.png
```

If sips cannot read the SVG directly (some macOS versions require a different approach), use this fallback: generate icon128.png by converting the SVG via a temporary HTML Canvas render using Node. The SVG path data in veil-icon.svg contains the shape — use the `--resampleWidth` flag first and only fall back if it produces an error.

After generation, verify file sizes are non-zero and confirm the files are valid PNGs.
  </action>
  <verify>
    <automated>node -e "
const fs = require('fs');
const sizes = [[16,'icon16.png'],[48,'icon48.png'],[128,'icon128.png']];
for (const [, name] of sizes) {
  const p = './extension/icons/' + name;
  const stat = fs.statSync(p);
  console.assert(stat.size > 100, name + ' too small, may be corrupt: ' + stat.size + ' bytes');
  const buf = fs.readFileSync(p);
  // PNG magic bytes: 0x89 0x50 0x4E 0x47
  console.assert(buf[0] === 0x89 && buf[1] === 0x50 && buf[2] === 0x4E && buf[3] === 0x47, name + ' is not a valid PNG');
}
console.log('All icon PNGs valid');
"</automated>
