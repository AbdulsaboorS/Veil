<task type="checkpoint:human-verify" gate="blocking">
  <name>Verify: Crunchyroll subtitle network intercept pipeline works end-to-end</name>
  <what-built>
    subtitle-interceptor.js — a MAIN world content script that overrides window.fetch, intercepts the Crunchyroll ASS subtitle file fetch (URL matching *.ass), parses cues (H:MM:SS.cs timestamps, {tag} stripping), and posts the cue list to the isolated world via window.postMessage({ type: '__VEIL_SUBTITLE_INTERCEPTED__', cues }). content.js receives the postMessage, starts a 1-second polling loop against video.currentTime, finds active cues, and writes { platform: 'crunchyroll', lines, contextText, ... } to chrome.storage.local['veil_context']. sidepanel.js picks up the storage change reactively and dispatches a VEIL_CONTEXT postMessage to the React app. useInitFlow.ts merges the last 5 lines into the active session's context field as a CURRENT SCENE: block.
  </what-built>
  <how-to-verify>
    **Step 0 — Load the updated extension:**
    1. Go to chrome://extensions, find Veil, click the reload button (circular arrow icon).
    2. If the extension was previously loaded from a different directory, check that the path still points to the correct `extension/` folder.

    **Step 1 — Check CSP (critical — do this first):**
    3. Open a Crunchyroll watch page (e.g. any episode URL like https://www.crunchyroll.com/watch/XXXXX/episode-title).
    4. Open DevTools → Network tab → filter by "ass" or "subtitle".
    5. Confirm you can see an XHR/Fetch request to a URL ending in .ass — this is the subtitle file the player fetches.
    6. Open DevTools → Console tab → in the Crunchyroll tab (not the side panel), run:
       ```javascript
       window.__veil_interceptor_loaded
       ```
       Expected: `true`. If `undefined`, the MAIN world script did not load. See CSP Failure path below.

    **Step 2 — Confirm cues were intercepted:**
    7. In the Crunchyroll tab console, run:
       ```javascript
       // Listen for the next VEIL_SUBTITLE_INTERCEPTED message (fire this before subtitles load, or seek back)
       window.addEventListener('message', e => { if (e.data?.type === '__VEIL_SUBTITLE_INTERCEPTED__') console.log('INTERCEPTED', e.data.cues.length, 'cues, sample:', e.data.cues.slice(0,2)); }, { once: true });
       ```
       Then seek the video back to the beginning (or pause and unpause). If subtitles are soft (ASS file), the fetch has already happened — check storage instead.
    8. In the Crunchyroll tab console, run:
       ```javascript
       chrome.storage.local.get('veil_context', console.log)
       ```
       Expected: `{ veil_context: { platform: 'crunchyroll', lines: [...], contextText: '...', updatedAt: '...', ... } }` with non-empty `lines` while a subtitle is on screen.
       If `lines` is empty but `contextText` is also empty, wait 5-10 seconds for the 1s sync loop to run and retry.

    **Step 3 — Confirm session context field is updated (SUB-01):**
    9. Open the Veil side panel. In DevTools for the side panel (right-click → Inspect on the side panel, or use chrome://inspect), run:
       ```javascript
       JSON.parse(localStorage.getItem('veil-sessions'))
         .find(s => s.sessionId === localStorage.getItem('veil-active-session'))
         ?.context
       ```
       Expected: String containing `"CURRENT SCENE:"` followed by subtitle lines.
    10. Wait 10 more seconds, re-run. Expected: CURRENT SCENE block has updated lines; recap summary above it is unchanged.

    **Step 4 — Confirm no re-detection (SUB-03):**
    11. While subtitles are playing, watch the side panel DevTools console.
    12. Expected: No `[Veil] phase: detecting` or `[Veil] phase: resolving` log lines after the initial `ready` state.

    **Step 5 — Confirm model awareness (SUB-02):**
    13. While a distinctive subtitle is visible on screen, ask in Veil chat: "What is happening right now?"
    14. Expected: Model response references the on-screen dialogue, not just the general episode recap.

    ---

    **CSP Failure path (if `window.__veil_interceptor_loaded` is `undefined`):**
    Crunchyroll's Content-Security-Policy may block MAIN world injection. To diagnose:
    - DevTools → Console → filter for "content security policy" errors.
    - DevTools → Network → click on the main document request → Response Headers → look for `content-security-policy` header.
    - If CSP is blocking the script, note the exact policy header value and report it so the next session can address it (e.g. via manifest-declared exception or alternative approach).
    - This is a known risk documented in the research file (02-RESEARCH-crunchyroll.md Q9 Pitfall 4). Statically-declared content scripts in MV3 are supposed to be exempt from page CSP — if it is still blocked, report as a Chrome version or policy edge case.

    **Hardsubbed content path (if veil_context.lines is empty after Step 2):**
    - If the Network tab shows NO .ass fetch (no URL ending in .ass at all), the content is hardsubbed — no soft subtitle file is fetched. The intercept cannot work for hardsubbed streams.
    - Note which show/episode was tested and whether it is hardsubbed.
    - Try a different show (e.g. Jujutsu Kaisen S1 or Attack on Titan) to confirm whether the issue is per-show or universal.
  </how-to-verify>
