When content.js captures subtitle lines and writes them to chrome.storage.local['veil_context'], sidepanel.js fires a VEIL_CONTEXT postMessage, and the listener merges the last 5 lines into the active session's context field as a "CURRENT SCENE:" block — without overwriting the recap summary and without triggering re-detection.
  </what-built>
  <how-to-verify>
    Load the updated extension in Chrome (chrome://extensions → reload the Veil extension after the 02-01 build).

    **SUB-01 — Subtitle lines reach session context:**
    1. Open a Crunchyroll or Netflix episode with subtitles enabled and actively playing.
    2. Wait ~5 seconds for subtitles to appear and the 3-second debounce to fire.
    3. In DevTools console on the side panel, run:
       ```javascript
       JSON.parse(localStorage.getItem('veil-sessions'))
         .find(s => s.sessionId === localStorage.getItem('veil-active-session'))
         ?.context
       ```
    4. Confirm the output contains "CURRENT SCENE:" followed by subtitle lines.
    5. Wait another ~10 seconds, run the command again — confirm the CURRENT SCENE block updates (new lines) without the recap summary being erased.
